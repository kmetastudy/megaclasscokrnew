# 메가클래스 문항 유형 가이드

## 개요

메가클래스 플랫폼에서 렌더링되는 문항은 총 **14가지 유형** + 기본 폴백으로 구분됩니다.
각 문항은 `ContentType.type_name` 값에 따라 학생 화면에서 서로 다른 템플릿과 JavaScript로 렌더링됩니다.

### 렌더링 흐름

```
slide_view.html
  ├── content_type.type_name 확인
  ├── 해당 유형의 템플릿 include (templates/student/slides/)
  ├── 공통 JS 로드 (scripts/common.html)
  └── 유형별 전용 JS 로드 (scripts/{type}.html)
```

### 데이터 모델 구조

| 모델 | 필드 | 역할 |
|------|------|------|
| `ContentType` | `type_name` | 문항 유형 식별자 |
| `Contents` | `page` | HTML 형식 콘텐츠 본문 |
| `Contents` | `answer` | 정답 데이터 (JSON 문자열) |
| `Contents` | `meta_data` | 메타데이터 (JSON) |
| `StudentAnswer` | `answer` | 학생 제출 답안 (JSON) |
| `StudentAnswer` | `is_correct` | 정답 여부 |
| `StudentAnswer` | `score` | 점수 (0~100) |
| `StudentProgress` | `is_completed` | 완료 여부 |

## 1. `multiple-choice` - 객관식 (단일 선택)

| 항목 | 내용 |
|------|------|
| **한글명** | 객관식 문제 |
| **템플릿** | `slides/multiple_choice.html` |
| **JS** | `scripts/choice_questions.html` (공통) |
| **채점 핸들러** | `handle_simple_answer()` |
| **자동 채점** | O |

### 특징
- 여러 보기 중 **하나만** 선택하는 기본 객관식
- HTML 기반 콘텐츠(`page` 필드)를 그대로 렌더링
- 정답은 `content.answer`에 단일 값으로 저장
- 학생 답안과 정답을 문자열 비교하여 채점
- 정답 시 100점, 오답 시 0점

### 정답 데이터 형식
```json
"1"
```

### 학생 답안 저장 형식
```json
{
  "selected_answer": "1",
  "correct_answer": "1"
}
```

## 2. `multi-choice` - 다중 선택

| 항목 | 내용 |
|------|------|
| **한글명** | 다중 선택 문제 |
| **템플릿** | `slides/multi_choice.html` |
| **JS** | `scripts/choice_questions.html` (공통) |
| **채점 핸들러** | 전용 분기 (`check_answer_0702`) |
| **자동 채점** | O |

### 특징
- 여러 보기 중 **복수 선택** 가능
- 학생 답안은 JSON 배열로 전송
- 정답은 콤마 구분 문자열 (예: `"1,2,3"`)로 저장 후 리스트로 변환
- 양쪽 모두 **정렬 후 비교**하여 순서 무관하게 채점
- 모든 선택이 일치해야 정답 (부분 점수 없음)

### 정답 데이터 형식
```json
"1,2,3"
```

### 학생 답안 저장 형식
```json
{
  "selected_answers": ["1", "2", "3"],
  "correct_answers": ["1", "2", "3"],
  "is_multiple": true
}
```

## 4. `multi-input` - 다중 입력

| 항목 | 내용 |
|------|------|
| **한글명** | 다중 입력 문제 |
| **템플릿** | `slides/multi_input.html` |
| **JS** | `scripts/multi_input.html` (전용) |
| **채점 핸들러** | 전용 분기 (`check_answer_0702`) |
| **자동 채점** | O |

### 특징
- **여러 개의 입력 필드**에 각각 답을 입력하는 문항
- 각 입력 필드별로 **개별 채점** 수행
- **부분 점수** 지원 (정답 개수 / 전체 개수 * 100)
- 정답은 `{"answer": {"key1": "값1", "key2": "값2"}}` 형태의 JSON으로 저장
- 모든 문항이 맞아야 100점 (비례 점수 계산)

### 정답 데이터 형식
```json
{
  "answer": {
    "input1": "정답1",
    "input2": "정답2",
    "input3": "정답3"
  }
}
```

### 학생 답안 저장 형식
```json
{
  "submitted_answers": {
    "input1": "학생답1",
    "input2": "학생답2",
    "input3": "학생답3"
  },
  "results": {
    "input1": {"is_correct": true, "user_answer": "학생답1"},
    "input2": {"is_correct": false, "user_answer": "학생답2"},
    "input3": {"is_correct": true, "user_answer": "학생답3"}
  }
}
```

```json
{
  "answer": "1",
  "solution": "해설 내용"
}
```

### 학생 답안 저장 형식
```json
{
  "selected_answer": "1",
  "correct_answer": "1",
  "solution": "해설 내용",
  "question_type": "ox-quiz"
}
```

### UI 구성요소
- 동적 배경 파티클 (`floating-particles`)
- 장식 이모지
- O/X 대형 버튼
- 정답/오답 피드백 메시지
- 결과 애니메이션 컨테이너

## 8. `drag` - 드래그앤드롭

| 항목 | 내용 |
|------|------|
| **한글명** | 드래그앤드롭 퀴즈 |
| **템플릿** | `slides/drag_quiz.html` |
| **JS** | `scripts/choice_questions.html` (공통) + `scripts/drag_quiz.html` (전용) |
| **채점 핸들러** | `handle_drag_answer()` |
| **자동 채점** | O |

### 특징
- 아이템을 **드래그하여 정해진 위치에 놓는** 인터랙티브 문항
- **두 가지 모드** 지원:
  - `sort` 모드: 아이템을 올바른 순서로 정렬
  - `match` 모드: 아이템을 올바른 대상에 매칭
- 드래그 아이템 컨테이너, 정렬 컨테이너, 매칭 컨테이너가 별도로 존재
- 답안은 JSON 객체로 전송 (매칭 상태)
- 정답 데이터에 `solution`(해설) 포함

### 정답 데이터 형식
```json
{
  "answer": {
    "slot1": "item_a",
    "slot2": "item_b"
  },
  "solution": "해설 내용"
}
```

### UI 구성요소
- `drag-items-container`: 드래그할 아이템 목록
- `sort-container`: 정렬 모드 드롭 영역 (sort 모드 전용)
- `match-container`: 매칭 모드 행 영역 (match 모드 전용)
- 정답/오답 피드백
- 파티클 애니메이션

## 12. `physical_record` - 체육 기록

| 항목 | 내용 |
|------|------|
| **한글명** | 체육 기록 |
| **템플릿** | `slides/physical_record.html` |
| **JS** | `scripts/physical_record.html` (전용) |
| **채점 핸들러** | `StudentPhysicalResult` 모델 사용 |
| **자동 채점** | X (기록 저장 방식) |

### 특징
- **체육 활동 기록**을 저장하는 특수 문항
- 일반 `StudentAnswer`가 아닌 **`StudentPhysicalResult` 전용 모델** 사용
- 기록 결과는 `JSONField`로 저장 (다양한 형태의 기록 수용)
- 기록자(`writer`) 정보 별도 저장
- `PhysicalResultType` 모델로 기록 유형 관리

### 데이터 모델 (`StudentPhysicalResult`)
| 필드 | 타입 | 설명 |
|------|------|------|
| `student` | FK(Student) | 학생 |
| `slide` | FK(ChasiSlide) | 슬라이드 |
| `record` | JSONField | 기록 결과 |
| `writer` | CharField | 기록자 |
| `score` | FloatField | 점수 |
| `submitted_at` | DateTimeField | 제출 시각 |

- 2주간 6가지 건강 약속 실천 기록
- 예시: 바른 자세, 규칙적 운동, 바른 식습관 등

### UI 구성
- 6가지 건강 약속 목록
- "실천 기록 시작하기" 버튼 (외부 링크)
- 그라데이션 배경의 정보 카드

|------|------|

| **채점 핸들러** | 없음 |
| **자동 채점** | X |

### 특징
- 위 14가지 유형에 해당하지 않는 경우의 **폴백(fallback) 렌더링**
- HTML 콘텐츠(`page`)를 그대로 출력
- 인터랙션이나 채점 기능 없음
- 단순 정보 전달용 슬라이드

## JS 스크립트 로딩 구조

```
slide_view.html
│
├── scripts/common.html (모든 유형 공통)

├── 그룹 1: 공통 choice 스크립트 로드 대상

│       ├── drag          → scripts/drag_quiz.html

│
├── 그룹 2: 독립 스크립트
│   ├── multi-input      → scripts/multi_input.html
```

## 채점 흐름도

```
학생 답안 제출 (AJAX POST)
│
├── ox-quiz        → handle_ox_quiz_answer()      → 문자열 비교
├── selection      → handle_selection_answer()     → 문자열 비교 + 해설
├── choice         → handle_choice_answer()        → handle_selection_answer() 위임
├── multiple-choice→ handle_simple_answer()        → 문자열 비교
├── short-answer   → handle_simple_answer()        → 문자열 비교
├── multi-choice   → check_answer_0702() 분기      → 리스트 정렬 비교
├── multi-input    → check_answer_0702() 분기      → 개별 필드별 비교 (부분점수)
├── drag           → handle_drag_answer()          → JSON 객체 비교
├── line_matching  → handle_line_matching_answer()  → JSON 연결 비교
├── one_shot_submit→ 전용 뷰                       → 제출 즉시 100점
├── physical_record→ StudentPhysicalResult 저장     → 점수 없음
├── rolling        → 외부 앱 처리                  → rolling 앱
└── take-action    → 외부 앱 처리                  → app_home 앱
```
